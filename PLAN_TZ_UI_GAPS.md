# План закрытия UI-пробелов по ALPHABASE.sale

Источник истины: `ALPHABASE_sale_final_master_document.docx`.

Ограничение: выполняем только верстку (без backend/business logic интеграций).

## Статус плана

- Утверждено к работе.
- Формат: задачи по приоритету (P1 -> P3).

## P1 — критично закрыть в первую очередь

### TASK-01: Виджеты рабочего стола из полного реестра ТЗ

Добавить недостающие виджеты (раздел 8.1 / 8.3):

- `Сделки под контролем`
- `Лиды без назначения`
- `Лиды с нарушением SLA`
- `Объекты без активности`
- `Сделки в зоне риска`

DoD (верстка):

- Каждый виджет имеет UI-состояния: empty / normal / warning.
- Виджеты встроены в текущий layout рабочего стола.
- Видимость по ролям отражена на уровне UI.

### TASK-02: Отдельная панель "Регистрация" в модуле "Новостройки"

Сейчас контур объединен как "Брони / Регистрации", по ТЗ нужна отдельная панель (раздел 10.15, 13.4).

DoD (верстка):

- Добавлен отдельный route под `Панель управления регистрацией`.
- В хабе `Новостройки` есть отдельная карточка `Регистрация`.
- Есть UI-скелет: реестр регистраций + карточка регистрации.

## P2 — важные структурные доработки

### TASK-03: Разделить отчеты "По менеджеру" и "По команде"

Сейчас агрегировано в один экран KPI, по ТЗ это два самостоятельных отчета (12.4 и 12.5).

DoD (верстка):

- Отдельные страницы:
  - `Отчет по менеджеру`
  - `Отчет по команде`
- На каждой странице отдельный шаблон: KPI-блок, таблица, фильтры/разрезы.
- Навигация из модулей `CRM` и `Команда` ведет на отдельные отчеты.

### TASK-04: Экран "Статусы агентства" (региональная статусная модель)

По ТЗ раздел 4 требует отдельного UI-контроля статусов по регионам.

DoD (верстка):

- Отдельная страница `Статусы агентства`.
- Табличный UI: регион, тип статуса, агентство-владелец статуса.
- UI-кнопки действий: назначить/снять статус.
- Состояния предупреждений для конфликтов уникальности статуса в регионе.

## P3 — расширение управляемости и навигации

### TASK-05: Единый "Реестр отчетов"

По ТЗ есть целостный реестр из 12 отчетов (раздел 11), нужен отдельный вход.

DoD (верстка):

- Отдельная страница `Реестр отчетов`.
- 12 карточек отчетов из ТЗ с переходами.
- Группировка по доменам: Лиды / CRM / Объекты / MLS / Финансы / Сообщество.

### TASK-06: Экран "Филиалы" в оргслое

Сущность `Филиал` есть в ТЗ (14.1), явного отдельного UI-контура сейчас нет.

DoD (верстка):

- Страница `Филиалы` в модуле `Команда` или `Настройки`.
- UI-скелет: список филиалов, карточка филиала, привязки команд/сотрудников.

## Примечание по качеству

Каждая задача должна сохранять текущий визуальный стиль проекта:

- существующие токены и переменные темы;
- единый стиль карточек, таблиц, фильтров, статусов и бейджей;
- согласованность с текущим layout `Dashboard` и `ModuleHub`.
